## 1. 集群环境初始化
将当前主机初始化为 Swarm 集群的管理节点（Manager）。

```bash
# 初始化 Swarm 集群
# 注意：若服务器存在多块网卡，建议使用 --advertise-addr 指定内网 IP
docker swarm init
```

## 2. 跨主机 Overlay 网络创建

创建一个覆盖网络（Overlay Network），实现不同物理节点间容器的二层互通。

```bash
# 创建支持外部挂载的 Overlay 网络
# -d overlay: 指定驱动为覆盖网络模式
# --attachable: 关键参数，允许非 Service 容器（如调试工具）手动接入此网络
docker network create -d overlay --attachable swarm-overlay-network
```

## 3. 网络连通性验证（全集群预热）

通过部署全集群模式（Global Mode）的服务，强制所有节点拉取镜像并建立通信链路。

### 步骤 A：镜像预热

```bash
# 在各节点预拉取基础测试镜像，确保服务启动效率
docker pull busybox
```

### 步骤 B：部署全局验证服务
```bash
# 在集群内所有可用节点强制部署容器
docker service create \
  -td \
  --name busybox-global \
  --mode global \
  --network swarm-overlay-network \
  busybox
```

### 步骤 C：资源清理
验证网络状态（`docker service ps busybox-global`）正常后，清理测试资源。
```bash
# 移除测试服务
docker service rm busybox-global
```

## 安全与优化建议

| 优化项 | 建议操作 | 目的 |
| --- | --- | --- |
| **端口开放** | 确保防火墙开启 `2377/tcp`, `7946/tcp/udp`, `4789/udp` | 保证集群节点心跳与数据平面互通 |
| **传输加密** | 创建网络时增加 `--opt encrypted` | 为跨主机流量启用 IPSEC 加密（推荐涉及医疗数据时开启） |
| **命名规范** | 建议将 `swarm-overlay-network` 替换为业务相关名称 | 提高生产环境资源辨识度 |
